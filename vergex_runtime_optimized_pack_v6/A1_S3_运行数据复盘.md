# A1 / S3 运行数据复盘与升级结论

## 采样范围
- A1_AI500_双周期趋势回撤_PLUS：16 个运行周期
- S3_NetflowTop_资金流顺势_PRO：16 个运行周期

## A1 为什么没有开单
- 这批样本里，A1 的总交易数持续为 **0**，账户权益和已用保证金都没有变化。
- AI500 候选高度集中在：ZECUSDT × 15, RAVEUSDT × 15, TRADOORUSDT × 14
- 候选池明显偏向“已经很强、但不一定给舒服入场”的币，导致原始 A1 的单一 pullback 逻辑长期 WAIT。

### 常见阻塞原因
| 原因 | 出现周期数 |
|---|---:|
| 过度延伸/抛物线 | 16 |
| 需要更深回撤 | 16 |
| 需要突破/回收确认 | 16 |
| 15m 区间/震荡太多 | 16 |
| 流向/OI 不够一致 | 16 |
| 缺少 BTC/ETH 背景时更保守 | 5 |

### 修复方向
1. 从 **只做回撤** 升级为 **回撤 + 高位整理突破回收** 双 archetype。  
2. 从 **完美对齐** 改成 **实用对齐**：OI / netflow 允许一项支持、一项中性。  
3. BTC / ETH 背景缺失时不再自动整体 WAIT，只降低激进度。  
4. 当前 source universe 里只要有一只满足条件，就优先做最优解，而不是因为其他候选混乱就整体 WAIT。  

## S3 为什么稳定
- S3 这批样本的总权益大致位于 **105.61 ~ 110.04**。
- 区间收益率大致在 **5.05% ~ 7.80%**。
- 它更像“顺势持有 + 管仓”的策略，而不是频繁试错型策略。

### S3 当前最大问题
- 平均保证金使用率约 **70.0%**，后半段明显偏高。
- 交易里出现了 BTCUSDC / HYPEUSDC，而 flow 参考会更多来自对应底层资产的 USDT 数据，因此需要显式写入 **quote-pair proxy** 规则。

### 修复方向
1. 明确允许用底层资产的 USDT flow/OI 作为 USDC 交易对的代理，但必须再看实际交易对价格结构。  
2. 新增 **margin regime**：>65% 先管仓，不要继续叠加相关仓位。  
3. 盈利持仓运行较久后，主动把止损从原始宽止损往 15m / 1h 结构收紧。  

## 对 v5 全策略的统一升级
- 非 Custom source：统一解释为 **source universe + Excluded Coins 黑名单**。  
- 所有趋势策略都支持 **pullback + breakout/retest/reclaim** 两类开仓。  
- 所有策略保留 WAIT，但必须给触发位。  
- 所有策略都加入保证金分层逻辑。
