# 公开资料依据与实操边界说明

本策略包的逻辑来源分为三层：

## 1. VergeX 官方公开文档
用于确认：
- Studio 创建策略的基本流程
- Strategy Engine 的执行链路
- Prompt 是策略核心组成部分
- 风控、杠杆、决策周期与市场数据的关系

## 2. NoFx 开源资料
用于理解这类 Agentic / AI 策略系统常见输入与约束：
- K-Line / EMA / MACD / RSI / ATR
- OI / 资金流 / OI changes
- 候选池筛选与结构化 prompt
- 多数周期允许 WAIT / HOLD
- 风险收益比至少 1:3 的保守思路

## 3. 你的实际 UI 与实操反馈
用于修正更贴近产品现实的部分：
- 非 Custom source 不支持正向白名单
- Excluded Coins 是黑名单
- Max Account Leverage 当前 UI 更适合填整数
- Run Test 可能与实际运行效果不完全一致，因此 Prompt 必须写得更硬、更明确

## 适用边界
- 本包适合：希望在 VergeX Studio 从 0 到 1 搭建可小仓实盘策略的人
- 不适合：把短期收益截图当成长期可复制结果的人
- 所有策略都应该先测试、再小仓、再逐步调整

## 最重要提醒
- 稳定度 / 工程置信度不是收益保证
- 高频策略和反转策略永远更脆弱
- 真正让策略可落地的，通常不是更复杂，而是：
  - source 与风格匹配
  - 时间框架匹配
  - Prompt 写清楚不交易条件
  - 风险参数足够保守
